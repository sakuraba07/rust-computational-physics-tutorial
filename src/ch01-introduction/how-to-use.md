# 本書の使い方

本書を最大限に活用し、計算物理学とRustプログラミングのスキルを効率よく習得するためのガイドです。本書は、理論を学び、コードを書き、結果を考察するというサイクルを繰り返すことで、実践的な能力が身につくように設計されています。

## 学習の進め方

本書の読み進め方には、主に2つのスタイルがあります。

### 1. 基本的な進め方：通読

計算物理学や数値計算の全体像を体系的に学びたい方は、第1章から順番に読み進めることを強く推奨します。前の章で説明した概念や実装が、後の章の基礎となっていることが多いため、最もスムーズに理解を深めることができます。

### 2. 目的別の進め方：興味のある分野から

特定の物理分野のシミュレーションに早く挑戦したい場合は、[はじめに](../index.html)で紹介した「目的別の学習パス」を参考に、必要な章だけをピックアップして学ぶことも可能です。

例えば、「統計力学シミュレーション」に興味があるなら、以下の順で読み進めるのが効率的です。

> 第1章 → 第2章 → 第9章 → 第12章

ただし、どのパスを選ぶ場合でも、 **第1部（基礎編）** は本書全体の共通言語となるため、一度は目を通しておくことをお勧めします。

## サンプルコードの実践ガイド

本書の核心は、実際にコードを書いて動かしてみる「実践」にあります。単にコードを読むだけでなく、ぜひご自身の手で入力し、実行し、そして「実験」してみてください。

### コードの実行手順

本書のサンプルコードは、以下の手順で実行することを想定しています。

1. **新しいプロジェクトを作成する**
   ターミナルで `cargo new` コマンドを使い、演習用のプロジェクトを作成します。

   ```bash
   cargo new physics-exercise
   cd physics-exercise
   ```

2. **コードを貼り付ける**
   お好みのエディタで `src/main.rs` を開き、本書に掲載されているコード例をコピー＆ペーストします。

3. **依存関係を追加する**
   `Cargo.toml` ファイルを開き、コードが必要とするクレートを `[dependencies]` セクションに追加します。バージョンは本書の表記に合わせるのが確実です。

   ```toml
   [dependencies]
   plotters = "0.3"
   # 他に必要なクレートがあれば追記
   ```

4. **コンパイルと実行**
   ターミナルで `cargo run` を実行します。Cargoが自動的に依存クレートをダウンロードし、コードをコンパイルして実行します。

   ```bash
   cargo run
   ```

### コードで「実験」しよう

コードが正しく動くことを確認したら、ぜひあなた自身でコードを改造し、「実験」をしてみてください。これが、理解を深める上で最も効果的な方法です。

- **パラメータを変えてみる**: シミュレーションのステップ数、物理定数、初期値などを変更すると、結果がどのように変わるか観察してみましょう。
- **別の題材に応用する**: `sin(x)` をプロットする代わりに、別の関数（`cos(x)` やガウス関数など）をプロットしてみましょう。
- **エラーから学ぶ**: エラーは最高の学習機会です。コンパイラが示すメッセージを注意深く読み、問題の原因を突き止めるプロセスを通じて、Rustへの理解が格段に深まります。

もしコードがうまく動かない場合や、完成形を確認したい場合は、本書の公式リポジトリにあるコードを参考にしてください。
（ただし、まずはご自身で解決策を探求することをお勧めします！）

## 各章の構成とヒント

本書の多くの章や節は、概ね以下の流れで構成されています。

1. **理論的背景**: トピックの背後にある物理的・数学的な概念を説明します。
2. **実装**: Rustでアルゴリズムを実装します。コードの重要な部分には解説を加えています。
3. **実行と考察**: コードを実行し、得られた結果（数値やグラフ）を元に、アルゴリズムの特性や物理的な意味を考察します。

章の最後には、学んだ知識をさらに深めるための **「発展課題」** を設けている場合があります。これらは必ずしも唯一の正解があるわけではなく、読者の皆さんが自由に探求するためのヒントです。ぜひ挑戦してみてください。

## 本書の表記ルール

本書では、情報を分かりやすく伝えるために、いくつかの特別な表記を使用しています。

> [!NOTE]
> **NOTE (補足)**
>
> 補足情報や、関連する豆知識などを記載します。

> [!IMPORTANT]
> **IMPORTANT (重要)**
>
> 特に重要な概念や、必ず覚えておいてほしい事柄を示します。

> [!WARNING]
> **WARNING (警告)**
>
> 注意しないとエラーや予期せぬ結果に繋がる可能性がある、重要な注意点を示します。
